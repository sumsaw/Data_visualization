Ford Go Bike System Data¶

by Sumeet Sawant¶

Preliminary Wrangling¶

This data set includes information about individual rides made in a bike-sharing system covering the greater San Francisco Bay area.

My aim here is to see how the ride duration in seconds is related or affected by other variables in the dataset

Describing the dataset .

I see that there are 2383421 rows and 16 columns the columns namly descibe

bike id: Unique id given to each bike
duration-sec : Time of each ride
End station id and start station id : Unique id given to each station
end_station_latitude and end_station_longtitude : latitude and longitude of each end station
start_station_latitude and start_station_latitude: latitude and longitude of each start station

end_station_name: Name of each end station
start_station_name: Name of each start station
start_time: Date and Time when ride started
end_time: Date and Time when ride ended
user_type: Customer has yearly pass and subscriber has a day pass or 3 day pass
memeber_gender: Gender identified by each user
member_birth_year: Year of birth for each user
bike_share_for_all_trips: To share bike info with the service provider

My main aim in this project is to see how the time-duration of the trip varies with other variables provided in the dataset as the time of the trip is directly proportional to the fare earned
